Highlight #1 Version Control

Version control is a system that records changes to a file or set of files over time so that you can recall specific versions later. A Version Control System (VCS) is important if a web or graphic designer wants to keep every version of an image or a layout. It lets you change some files back or the whole project back to its previous state. If you use a Version Control System and mess up on some things or lose files, it is easy to recover and fix. There are three types of Version Control Systems: Local, Central, and Distributed Version Control Systems.

Highlight #2 Git

Git is a free and open source distributed version control system designed to handle very small and large projects. It is primarily used for software development. Instead of Git "storing" your files, it basically takes a picture of your work and references back to the snapshot. Git's data is more like a "stream of snapshots." Pretty much every operation is local, so you never need another computer or network. Git has three main states that your files can reside in: committed, modified, and staged. Some basic git commands are "git init," "git add --all," "git commit -am 'Initial commit'", and "git log." Some remote commands are "git remote," "git pull," and "git push."

Highlight #3 Getting a Git Repository

There are two main ways to get a Git project. The first takes an existing project or directory and imports it into Git. The second clones an existing Git repository from another server. First you need to go to the project's directory if you are trying to track an existing Git project. This makes a new subdirectory called ".git" that contains all of the necessary repository files also known as a Git repository skeleton. Nothing in the project is tracked yet. If you want to get a copy of an existing Git repository, the command needed is "git clone."

Highlight #4 Recording Changes to the Repository

The main tool you use to determine which files are in which state is the "git status" command. In order to begin tracking a new file, you use the "git add" command. New files that aren’t tracked have a "??" next to them, new files that have been added to the staging area have an "A," modified files have an "M" and so on. If you want to ignore a file, you can create a file listing patterns to match them named ".gitignore". There are many more commands to view staged and unstaged changes such as the "git diff" command.

Highlight #5 A Short History of Git

Since its birth in 2005, Git has evolved and matured to be easy to use and yet retain these initial qualities. It’s incredibly fast, it’s very efficient with large projects, and it has an incredible branching system for non-linear development. Some of the goals of the new system were speed, simple design, strong support for non-linear development (thousands of parallel branches), fully distributed, and being able to handle large projects like the Linux kernel efficiently (speed and data size). The Linux development community (and in particular Linus Torvalds, the creator of Linux) developed Git based on some of the lessons they learned while using BitKeeper.
